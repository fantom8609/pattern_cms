1) внутри папки core находятся классы нашего движка
2) внутри service будут провайдеры, там будут абстрактные классы, которые будут инициализировать все наши классы,
которые будут храниться в папке core и передаваться в DI контейнер, чтобы они были доступны во всех местах нашей
системы.
3) ди контейнер это паттерн для внедрения зависимостей
4) с помощью неймспейсов мы можем потом где угодно подключить данный файл use Engine\DI
 или создать экземпляр его класса. Например: new Engine\DI\DI()
5) RewriteCond %{REQUEST_FILENAME} !-d   // реврайт будет обрабатывать все запросы но только при условии
что у нас не существует этих папок
   RewriteCond %{REQUEST_FILENAME} !-f

6) RewriteRule ^(.*)$ index.php/$1 [L] //при любом запросе будет открываться index.php, а строку запроса
будет добавлять в массив сервер REQUEST_URL
